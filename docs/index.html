Save the Novel

project architecture — the reader speaks

A website to gather and broadcast the experience of reading — not for information, but for connection, belonging, empathy, nuance. Personal essays by readers and authors about the novels that changed their lives, each linked to a favorite independent bookstore.

The Problem
Education → Testing
US educators optimize for standardized assessments. Literature is reduced to excerpts and passages.
Comprehension-Only
A generation educated to read for information, without meaningful connection to the text.
AI Summarizes
If AI can digest and summarize instantly, why would that generation pick up a novel at all?
the real value can't be quantified
The Insight
Empathy
Tolerance for ambiguity
Holding contradictions
Connection & belonging
Readers know this. But if we can't demonstrate why we read novels, how can we ensure the future preservation of the craft?
let the reader speak
Core Content: The Essay

A personal letter to the author of the novel that changed the contributor's life. Long-form, untruncated, meant to be read slowly.

The Questions
Where were you when you read the novel that changed you?
Who were you with?
Who were you?
How were you changed?

recommended tech stack
Recommended Tech Stack
Astro (Static Site Generator)
Content-focused, outputs zero JS by default, excellent Markdown support, fast builds. Perfect for a reading-first site.
Decap CMS (Content Management)
Visual editor in the browser. Essays stored as Markdown files in Git. Free. No database required.
Netlify (Hosting)
Free tier, auto-deploys from Git, built-in forms, pairs naturally with Decap CMS. SSL included.
Buttondown (Newsletter)
Simple, Markdown-native email. Indie-spirited, affordable. One featured essay per month.
Plausible (Analytics)
Privacy-respecting, no cookies, lightweight. Track essay reads, bookstore clicks, signups.
GitHub (Source & Collaboration)
Version control for the site and all essay content. Decap CMS commits directly to the repo.
